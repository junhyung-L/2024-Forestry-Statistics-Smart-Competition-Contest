## 프로젝트 개요

- **참여 기간:** 2024.10.26 ~ 2024.11.22
- **역할:** 팀장/ 데이터 수집·정제(ETL) / 통계 분석·모델링 / 시각화 / 웹 서비스 기획(플랫폼 기능 정의)
- **툴 / 사용 기술:** `Python (pandas, numpy, matplotlib, geopandas)`중심 분석 및 GIS 기반 지도 시각화

    - 기존에는 토양·기후·판매 데이터가 분산되어 있어 예비 임업인의 탐색비용이 크다.
    - 공공데이터를 통합해 재배적지 추천과 가격·판매량 정보를 동시에 제공하는 구조가 합리적이라고 판단했다.

## 프로젝트 전체 회고

- Why(이유)
    - 임산물 재배와 관련된 의사결정에 필요한 토양·기후·시장 정보가 개별적으로 흩어져 있어, 
    재배 지역 선택 과정에서 높은 탐색 비용이 발생하고 있었다.
    - 특히 예비 임업인이나 소규모 생산자의 경우, 데이터를 종합적으로 해석할 수 있는 기준이 
    부재하다는 점이 구조적 문제로 인식되었다.
    - 이에 따라 단순한 통계 제공이 아닌, **실제 선택에 활용 가능한 형태로 정보를 재구성하는 것**을 분석의 출발점으로 설정했다.
- How(과정)
    - 먼저 데이터의 성격을 명확히 구분했다. 기후 변수는 연속형, 토양 변수는 범주형이라는 점에 착안해 분석 기법을 분리했다.
    - 통계 분석을 통해 1차적으로 의미 있는 후보 요인을 도출한 뒤, 머신러닝 모델을 활용해 
    중요도를 재검증하는 단계적 접근을 사용했다.
    - 데이터 결측, 지역 간 편차, 생산량 불균형 문제는 전처리 기준을 명확히 설정하고 분석 
    단위를 통일함으로써 보완했다.
    - 분석 결과는 수치 해석에 그치지 않고, 지도 시각화와 웹 서비스 구조로 연결해 해석 가능성을 높이고자 했다.
- What(결과)
    - 임산물별로 생산량에 영향을 미치는 주요 환경 요인을 구조적으로 정리하고, 이를 기반으로 재배 적합 지역을 제안할 수 있었다.
    - 분석 결과를 웹 서비스 형태로 구현하면서, 데이터 분석이 실제 의사결정 도구로 전환되는 
    과정을 경험했다.
    - 이 과정을 통해 분석 기법 선택의 중요성, 모델 결과 해석의 상대성, 그리고 데이터를 ‘설명’이 아닌 ‘활용’ 관점에서 바라보는 역량을 키울 수 있었던 것 같다.
